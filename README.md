CS-255 Portfolio

CS 255 Module Eight Journal – Portfolio Submission

This course focused on the DriverPass system, which is a web-based application designed to help students prepare for driving exams. The system provides practice tests, scheduling for driving lessons, and administrative tools for managing users and system data. The goal of the system is to improve student success rates by providing a centralized and accessible training platform.

Summary of the DriverPass Project

The DriverPass system was designed for a client who wanted an online platform to support driving test preparation. The system includes a customer-facing portal where students can take practice exams and schedule driving lessons, as well as an administrative dashboard for managing users, instructors, and system operations. The system also uses a relational database to store user accounts, test results, and scheduling information. Different user roles such as students, administrators, and instructors interact with the system in different ways based on their permissions.

What I did particularly well

I think I did well in breaking the system into clear components and representing how different users interact with it. My UML diagrams helped define how the system functions, especially the use case diagram which shows the main system interactions, and the activity diagrams which break down processes like booking a reservation and managing user accounts. I also did well in defining the technical requirements, including the use of cloud hosting, relational databases, and secure authentication systems with role-based access control.

If I could revise one part of my work

If I could revise one part of my work, I would improve the level of detail and consistency in my UML diagrams, especially the class diagram and sequence diagram. I would make sure that all relationships between system components are clearly labeled and that each diagram is fully aligned with the written requirements. This would make the system design easier to understand at a deeper technical level.

Interpreting user needs

I interpreted user needs by focusing on the goals of each user type in the system. Students needed access to practice exams and lesson scheduling, instructors needed a way to manage lesson assignments, and administrators needed control over users and system data. These needs were translated into system functions such as scheduling reservations, managing accounts, generating reports, and securely storing data. Understanding user needs is important because the entire system is built around solving real user problems, and if those needs are not correctly captured, the system will not be effective regardless of technical quality.

My approach to software design

My approach to software design begins with understanding the system requirements and identifying the main user roles. From there, I break the system into components and model how data flows between users, the application server, and the database. I then use UML diagrams to visually represent system behavior and structure. In future projects, I would continue using structured analysis, iterative refinement, and diagram-based design to ensure that system requirements are accurately translated into functional software designs.
